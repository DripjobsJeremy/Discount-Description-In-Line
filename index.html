📋 Dev Annotation — Inline Discount Description
  • Placement: Description renders directly below the discount label/amount row, inside the green container box.
  • Font: 11px, color #9ca3af, weight 600 — identical to former hint text.
  • 2-line clamp: CSS -webkit-line-clamp: 2 hard-caps display at 2 lines. Text truncates with ellipsis if it overflows.
  • Character limit (backend): Enforce max ~120 characters on the Apply Discount modal description field to prevent excessive truncation at the customer view.
  • Empty state: If no description is entered, the .discount-description element is not rendered — no extra padding or empty line appears.
  • Applies to: Both flat-dollar and percentage discount types.
  • PDF: Print description as a second line beneath the discount label in generated PDFs.
  • Edit flow: Clicking the green discount row re-opens the Apply Discount modal pre-filled with current values.
